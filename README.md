OneCart
Group food ordering for friends, hostels and teams — one shared cart, one Swiggy delivery, everyone pays their own share.

The Problem
Ordering food with a group on Swiggy today looks like this — someone shares their phone, everyone calls out their order while one person types, the bill goes under one name, and then comes the awkward "hey can you GPay me back" message.
OneCart fixes this.

What It Does
OneCart lets a group create a shared ordering session. Each person joins from their own device, picks their own dishes from a single restaurant, and pays individually. The orders are combined into one delivery to the same address, placed via Swiggy.
No shared logins. No one person footing the bill. No payment follow-ups.

Who It's For

Friend groups ordering in together
Hostel and PG residents splitting a meal
College classmates ordering to campus
Colleagues doing a team lunch
Swiggy One sharers — families or roommates on a shared membership


How It Works

Host creates a session, selects a restaurant, shares an invite link
Friends join on their own devices, browse the same menu, add to their own cart
Everyone checks out individually and pays their own amount via Swiggy
A single combined order is placed — one delivery, one address


Stack
React Native + Expo, Node.js on Railway, PostgreSQL, Supabase Auth, Swiggy MCP API
